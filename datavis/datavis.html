Seth Godin was quoted as saying, "Is something important because you measure it, or is it measured because it's important?"

Guided by this, let me make it clear: I'm doing this because I find it fun, not to put weight on anything that I graph. If in the process I gained a valuable insight on something, then I'm happy with the fact that I understand which works and which doesn't.



Following are the chart presentations I used. Accompanying examples found on the sidebar.

DASHBOARDS

Interactive dashboards using various Javascript libraries. Focus on a particular event by clicking and drag selecting different portions of the charts.

GEOMAP

Visualizing geographic features that give a glimpse on some world/ continent/ country wide trends.

FORCE LAYOUT DIAGRAM

Here's the equation: data + physics = awesome!

SCATTERPLOT

Similar to line graphs, scatterplots map quantitative data points. However, using this means deciding that individual points should not be directly connected with a line segment. Instead, this is used to extract a trend. A single dot on the chart represents a single data point.

SANKEY DIAGRAM

Sankey diagrams are a form of flow diagram where the width of the connectors are proportional to the quantity of flow. Common applications include visualisation of cost or energy or product transfers between the processes.

BAR GRAPH

A very common type of graph most suitable for visualising qualitative independent variable as there is no need for consistent distance between bars. With the bars gently in size, one can discern the patterns. Slopes are off limits here.

LINE GRAPH

Unlike scatterplots, the points can be connected by lines to impart how the independent variable changes in value over time or something else that happens. The connecting line shows a slope that can be visually interpreted relative to the slope of other line segment.
